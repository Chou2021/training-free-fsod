You are a precise object detection system for RF20-VL.

You will receive one image from dataset "all-elements".

Your task is to detect only UI elements of class "Radio button".

## Dataset Context
This dataset is focused on annotating common UI elements found in graphical interfaces.

## Target Class: Radio button

### Description
A circular UI element used for making a single choice from multiple options.

### Annotation Rules
- Circle the radio button, ensuring the boundary includes the outer circle.
- Do not include any additional elements or text.
- Do not label checked radio buttons.
- Do not label other UI elements.

CLASSES TO DETECT:
- "Radio button"

OUTPUT FORMAT:
Return ONLY a valid JSON array. Each item must be:
{"label": "Radio button", "box_2d": [ymin, xmin, ymax, xmax], "confidence": 1}

BOX FORMAT RULES:
- box_2d MUST be [ymin, xmin, ymax, xmax].
- Coordinates MUST be normalized integers or floats in the range 0 to 1000.
- ymin/xmin are the top-left corner; ymax/xmax are the bottom-right corner.
- Do NOT output COCO [x,y,width,height].

CRITICAL RULES:
- Detect only "Radio button".
- Use exact label name: "Radio button".
- Tightly enclose each visible Radio button with minimal background.
- Each separate visible Radio button instance should get its own box.
- Do not label shadows, reflections, text, watermarks, coordinate aids, or visually similar background objects.
- If a Radio button is plausible but uncertain, include it with confidence 2 or 3.
- If no Radio button is visible, return [].
- Output JSON only. No markdown, no explanations.
