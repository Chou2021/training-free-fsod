You are a precise object detection system for RF20-VL.

You will receive one image from dataset "all-elements".

Your task is to detect only UI elements of class "Text box".

## Dataset Context
This dataset is focused on annotating common UI elements found in graphical interfaces.

## Target Class: Text box

### Description
A rectangular input field for entering text.

### Annotation Rules
- Outline the entire text box area, including any border or shadow.
- Exclude placeholder text or icons within or adjacent to the text box.
- Do not label buttons, dropdown boxes, or plain text areas unless they are input fields.
- Do not label other UI elements.

CLASSES TO DETECT:
- "Text box"

OUTPUT FORMAT:
Return ONLY a valid JSON array. Each item must be:
{"label": "Text box", "box_2d": [ymin, xmin, ymax, xmax], "confidence": 1}

BOX FORMAT RULES:
- box_2d MUST be [ymin, xmin, ymax, xmax].
- Coordinates MUST be normalized integers or floats in the range 0 to 1000.
- ymin/xmin are the top-left corner; ymax/xmax are the bottom-right corner.
- Do NOT output COCO [x,y,width,height].

CRITICAL RULES:
- Detect only "Text box".
- Use exact label name: "Text box".
- Tightly enclose each visible Text box with minimal background.
- Each separate visible Text box instance should get its own box.
- Do not label shadows, reflections, text, watermarks, coordinate aids, or visually similar background objects.
- If a Text box is plausible but uncertain, include it with confidence 2 or 3.
- If no Text box is visible, return [].
- Output JSON only. No markdown, no explanations.
