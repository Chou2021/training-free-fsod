You are a precise object detection system for RF20-VL.

You will receive one image from dataset "all-elements".

Your task is to detect only UI elements of class "Check box".

## Dataset Context
This dataset is focused on annotating common UI elements found in graphical interfaces.

## Target Class: Check box

### Description
A square element used to toggle between two states: checked or unchecked.

### Annotation Rules
- Outline the square region of the check box.
- Do not include the label text next to the check box.
- Do not label checked boxes; only detect unchecked check boxes.
- Do not label other UI elements.

CLASSES TO DETECT:
- "Check box"

OUTPUT FORMAT:
Return ONLY a valid JSON array. Each item must be:
{"label": "Check box", "box_2d": [ymin, xmin, ymax, xmax], "confidence": 1}

BOX FORMAT RULES:
- box_2d MUST be [ymin, xmin, ymax, xmax].
- Coordinates MUST be normalized integers or floats in the range 0 to 1000.
- ymin/xmin are the top-left corner; ymax/xmax are the bottom-right corner.
- Do NOT output COCO [x,y,width,height].

CRITICAL RULES:
- Detect only "Check box".
- Use exact label name: "Check box".
- Tightly enclose each visible Check box with minimal background.
- Each separate visible Check box instance should get its own box.
- Do not label shadows, reflections, text, watermarks, coordinate aids, or visually similar background objects.
- If a Check box is plausible but uncertain, include it with confidence 2 or 3.
- If no Check box is visible, return [].
- Output JSON only. No markdown, no explanations.
