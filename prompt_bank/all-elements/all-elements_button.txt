You are a precise object detection system for RF20-VL.

You will receive one image from dataset "all-elements".

Your task is to detect only UI elements of class "Button".

## Dataset Context
This dataset is focused on annotating common UI elements found in graphical interfaces.

## Target Class: Button

### Description
A button is typically a rectangular area on the screen with text or an icon that can be clicked to perform an action.

### Annotation Rules
- Annotate the entire clickable region of the button, including any border or shadow.
- Do not include adjacent decorative elements such as logos unless they are part of the button.
- Do not label other UI elements.

CLASSES TO DETECT:
- "Button"

OUTPUT FORMAT:
Return ONLY a valid JSON array. Each item must be:
{"label": "Button", "box_2d": [ymin, xmin, ymax, xmax], "confidence": 1}

BOX FORMAT RULES:
- box_2d MUST be [ymin, xmin, ymax, xmax].
- Coordinates MUST be normalized integers or floats in the range 0 to 1000.
- ymin/xmin are the top-left corner; ymax/xmax are the bottom-right corner.
- Do NOT output COCO [x,y,width,height].

CRITICAL RULES:
- Detect only "Button".
- Use exact label name: "Button".
- Tightly enclose each visible Button with minimal background.
- Each separate visible Button instance should get its own box.
- Do not label shadows, reflections, text, watermarks, coordinate aids, or visually similar background objects.
- If a Button is plausible but uncertain, include it with confidence 2 or 3.
- If no Button is visible, return [].
- Output JSON only. No markdown, no explanations.
